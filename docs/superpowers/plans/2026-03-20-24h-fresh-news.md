# 24-Hour Fresh News Filter Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restrict Newscrux to processing and delivering only articles published within the last 24 hours.

**Architecture:** Add a single freshness-window configuration and thread `publishedAt` through queue persistence so the app can reject stale items at intake time and clean them out of pending queue states on each poll cycle. Keep the logic centralized in small helpers so fetch, queue, and runtime cleanup all follow the same definition of freshness.

**Tech Stack:** TypeScript, Node.js, existing RSS fetch pipeline, JSON queue persistence, Node test runner with `tsx`
